Note: this page is work in progress, as is the software described here. We are at the moment preparing for a very first beta version of Lab::Measurement. The old, "stable" relase (called Lab::VISA) can be found at http://www.labvisa.de/.

Lab::Measurement allows to perform test and measurement tasks with Perl scripts. It provides an interface to several instrumentation control backends, as e.g. Linux-GPIB or National Instruments' NI-VISA library. Dedicated instrument driver classes relieve the user from taking care for internal details and make data aquisition as easy as 

$voltage = $multimeter->read_voltage();

The Lab::Measurement software stack comprises four parts that are built on top of each other and provide increasing comfort. Measurement scripts can be based on any of these stages.

The lowest level is the device driver library or Perl binding. Lab::VISA, distributed here, makes the NI-VISA library accessible from Perl and thus allows to make any standard VISA call. This way, a wide variety of devices can be addressed. Installation of NI-VISA is straightforward on Windows but can be VERY cumbersome on recent Linux variants. Therefore, other independently developed drivers such as Linux-GPIB, the de-facto standard for addressing GPIB devices under Linux, can also be used; support for more backends is being developed.

As the backends vary in API details, each of these is encapsulated into a perl module of type Lab::Connection. Imagine a Connection as a bus, or a GPIB cable. All measurement devices on one cable share a Connection, passing commands and data to and from the computer. If your measurement script works under Linux with Lab::Connection::GPIB, it will also work with Lab::Connection::VISA by just instantiating a different Connection object.

The modules in the Lab::Instrument package make communication with instruments easier by silently handling the protocol overhead. Drivers for specific devices are included; more can easily be added to provide high-level functions. Package Lab::Tools is the highest abstraction layer and is intended to support writing better measurement scripts. The modules in this package offer means to log and plot data and its related meta information.

These packages together are referred to as the Lab::Measurement system. Designed to make data aquisition fun!

How to obtain

The packages are free software and can be downloaded from CPAN. Follow these links for

  • Lab::VISA,
  • Lab::Connection,
  • Lab::Instrument, and
  • Lab::Tools.
The source code archive is hosted at Gitorious, where you can also obtain the newest pre-release code and browse the version history. If you would like to contribute, just send us your patches, merge requests, ... :)

Documentation

Quite some documentation of Lab::Measurement (PDF format) is available. This documentation includes a tutorial on using Lab::VISA (outdated). Detailed installation instructions are provided as well. In addition, there's also a collection of back-end specific documentation and links.

There is a mailing list (lab-visa-users) set up for Lab::VISA. This mailing list is the right place to give feedback and ask for help.

Status

Although this software has been used for years in real world measurements by its developers, it remains work in progress. Please bear with us while we constantly improve code and documentation.

Lab::Measurement is currently developed and employed at nanophysics group, LMU München and mesoscopic physics group, Uni Regensburg. Users have reported further applications in academic and industrial r&d environments.

Authors and history

The Lab::VISA system was originally developed by Daniel Schröer and continued by Andreas K. Hüttel, Daniela Taubert, and Daniel Schröer. Most of the documentation was written by Daniel Schröer. In 2011, the code was refactored mostly by Florian Olbrich to include the Connection layer; subsequently the name of the entire package collection was changed to Lab::Measurement.
